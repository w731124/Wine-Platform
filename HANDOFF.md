### 技術細節
- `index.html`：導覽列 HTML 改用 `.tab-group` 包 `.tab-group-trigger` + `.tab-dropdown`（含多個 `.tab-dropdown-item`）的結構。
- `css/style.css`：新增 `.tab-group`／`.tab-dropdown`／`.tab-dropdown-item` 等樣式，沿用既有 CSS 變數（`--burg`／`--gold`／`--bg-card` 等），未新增行內樣式。
- `js/core.js`：新增 `toggleTabGroup()`、`selectTabFromGroup()`、點擊外部關閉下拉的全域監聽；**修正了既有的 `switchToPanel(name)`**（原本用於產區↔品種跳轉連結）——舊的比對邏輯在新的下拉結構下會誤比對到下拉項目而非群組觸發鈕，導致跳轉後頂層看不出目前在哪一類，動工時發現並一併修正。

### 驗證方式（新手法）
除了先前已知的 headless Chrome `--dump-dom`（只能驗證靜態渲染），這次發現 **`.click()` 是原生 DOM API，可以在不透過 CDP WebSocket 的情況下觸發真實 onclick handler**，因此寫了一個獨立測試頁載入真實 `js/core.js`，用 `.click()` 實際跑過7種互動情境（展開/收合、跨群組切換、獨立按鈕、`switchToPanel()` 連動、點外部關閉），全數驗證正確。另外也對正式 `index.html` 跑過 `--dump-dom` 確認無載入錯誤、先前功能未被波及。
